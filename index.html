Turtles, Dragons and the Lindenmayer System

The first thing I ever programmed was a Big Trak, which was this toy truck that could be programmed to follow a route (forward, back, turn left, forward again etc). So I'm regressing to childhood with this attempt to program a Turtle drawing tool in Javascript.

I started off looking at dragon curves, 'recursive nonintersecting curve whose name derives from its resemblance to a certain mythical creature'. There are a couple of methods for drawing approximating them and my first attempt used a formula found on the wikipedia page (and a little investigation into bitwise operators).

                    
                        for(var n = 1; n <= 1000; n++) {
                        turtle.forward(length);
                        if((((n & -n) << 1) & n) != 0) turtle.rotate(0.25);
                        else turtle.rotate(-0.25);
                    }

Lindenmayer Systems are an interesting method of defining patterns in the form of strings of characters that represent actions like 'move forward' or 'rotate'. The resulting string can be used to control the turtle. In general and L-System consists of:

  • An alphabet of symbols that can be used to make strings.
  • An initial 'axiom' string from which contruction begins.
  • A collection of rules that are repeatedly applied to the string, expanding it into some larger string of symbols.
  • A mechanism for translating string into geometric structure.

For example a dragon curve could be written in L-system as:

  • Axiom FX
  • Variables [X, Y]
  • Constants [F, +, -]
  • Rules:
  • X converts to X+YF+
  • Y converts to -FX-Y
  • F converts to Empty String (i.e. remove F)

Applying these rules would result in:

0: FX
1: X+YF+
2: X+YF++-FX-Y+
3: X+YF++-FX-Y++-X+YF+–FX-Y+
4: X+YF++-FX-Y++-X+YF+–FX-Y++-X+YF++-FX-Y+–X+YF+–FX-Y+
5: X+YF++-FX-Y++-X+YF+–FX-Y++-X+YF++-FX-Y+–X+YF+–FX-Y++-X+YF++-F
X-Y++-X+YF+–FX-Y+–X+YF++-FX-Y+–X+YF+–FX-Y+

This can be translated into a curve by applying the following:

  • F means draw forward
  • + means turn left (90 degrees)
  • - means turn right (90 degrees)

X and Y are necessary for building the L-System string but ignored when translating that to a geometric pattern.
